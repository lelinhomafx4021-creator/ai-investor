# 代码审查技能

当用户要求你审查代码时，请按照以下检查清单进行:

## 代码审查检查清单

### 1. 正确性

- 代码逻辑是否正确？
- 是否有明显的 bug？
- 边界条件是否处理？

### 2. 代码风格

- 命名是否清晰易懂？
- 代码结构是否合理？
- 是否有重复代码可以抽取？

### 3. 性能

- 是否有性能瓶颈？
- 数据库查询是否高效？
- 是否有 N+1 查询问题？

### 4. 安全性

- 是否有 SQL 注入风险？
- 敏感数据是否安全处理？
- 权限校验是否完善？

### 5. 可维护性

- 注释是否充分？
- 代码是否易于测试？
- 是否遵循 SOLID 原则？

## 反馈格式

审查完成后，请按以下格式输出:

1. **总体评价**: 一句话总结代码质量
2. **问题列表**: 按严重程度排序的问题清单
3. **改进建议**: 具体的代码改进建议
