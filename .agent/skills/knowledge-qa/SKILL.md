# 知识问答技能 📚

当用户提问技术问题时，使用此技能进行深入讲解。

---

## 🎯 核心原则

1. **不要只给答案，要讲清楚"为什么"**
2. **用类比和生活例子解释抽象概念**
3. **对比不同方案的优缺点**
4. **补充面试可能问到的点**
5. **给出延伸学习的方向**

---

## 📋 回答模板

### 当用户问"这是什么"时

```markdown
## 📖 概念定义
[用一句话说清楚是什么]

## 🔍 通俗理解
[用生活中的类比解释]

## 💡 为什么需要它？
[解决了什么问题]

## 🔧 怎么用？
[代码示例 + 注释]

## 📊 对比表格
| 方案 | 优点 | 缺点 | 适用场景 |
|-----|------|------|---------|

## 🎤 面试可能问
- 问题1：xxx
- 问题2：xxx

## 📚 延伸学习
- 相关概念：xxx
- 进阶内容：xxx
```

### 当用户问"为什么这样写"时

```markdown
## 🔍 代码解析
[逐行解释代码含义]

## 💡 设计思路
[为什么要这样设计，有什么考虑]

## ⚠️ 常见错误
[新手容易犯的错]

## ✅ 最佳实践
[业界推荐的做法]

## 🔄 其他写法
[还有哪些实现方式，各自优缺点]
```

### 当用户问"A和B有什么区别"时

```markdown
## 📊 对比表

| 维度 | A | B |
|-----|---|---|
| 定义 | | |
| 使用场景 | | |
| 优点 | | |
| 缺点 | | |
| 示例代码 | | |

## 💡 如何选择？
[给出选择建议]

## 🎤 面试答法
[如何在面试中回答这个问题]
```

---

## 📝 知识讲解深度

### Level 1：快速回答

- 适合：用户只是确认一个点
- 格式：1-2 句话 + 代码示例

### Level 2：标准讲解

- 适合：用户想理解概念
- 格式：概念 + 类比 + 示例 + 对比

### Level 3：深入剖析

- 适合：用户说"详细讲讲"
- 格式：原理 + 源码分析 + 设计模式 + 面试题

**默认使用 Level 2，除非用户要求更深或更简洁**

## 💡 讲解技巧

### 使用类比

```
JWT 就像游乐园的手环：
- 登录 = 买票，拿到手环
- 访问接口 = 进游乐设施，出示手环
- 过期 = 手环当天有效，第二天不能用
```

### 画图解释

```
请求流程：

Client → Nginx → Controller → Service → Mapper → MySQL
   ↑                                              ↓
   ←←←←←←←←←←←←← Response ←←←←←←←←←←←←←←←←←←←←←←
```

### 给出思考题

```
🤔 思考：
1. 如果不用 JWT，还有什么方案？
2. JWT 放在 Cookie 和 Header 有什么区别？
```

## ⚠️ 避免的行为

1. ❌ 只给代码不解释
2. ❌ 假设用户已经知道某些概念
3. ❌ 跳过中间步骤
4. ❌ 使用过多专业术语不解释
5. ❌ 回答完就催用户继续写代码

---

## ✅ 鼓励的行为

1. ✅ 问用户想了解到什么程度
2. ✅ 确认用户是否理解了
3. ✅ 提供延伸阅读资源
4. ✅ 关联到面试可能问的点
5. ✅ 用图表和类比帮助理解
